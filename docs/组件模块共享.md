## 组件模块共享
自定义组件间可通过模块联邦方式进行模块共享。

### 步骤 1：在组件 A 中导出模块

在 `customCmpA` 的 `neo.config.js` 中通过 `neoCommonModule.exports` 导出要共享的模块（对象写法需使用绝对路径）：

```javascript
// customCmpA 的 neo.config.js
const path = require('path');

module.exports = {
  neoCommonModule: {
    // 数组写法：导出第三方依赖模块
    // exports: ['xxModule'],
    // 对象写法：导出工程内某路径下的模块
    exports: {
      xxModuleA: path.resolve(__dirname, './src/components/xxModuleA'),
    },
  },
};
```

### 步骤 2：在组件 B 中声明远程依赖

在 `customCmpB` 的 `neo.config.js` 中声明依赖的组件名及要从外部剔除的模块名（`externals` 仅支持数组，且需与 `remoteDeps` 配合）：

```javascript
// customCmpB 的 neo.config.js
module.exports = {
  neoCommonModule: {
    remoteDeps: ['customCmpA'], // 远程依赖（自定义组件），表示当前自定义组件 B 会用到哪些自定义组件
    externals: ['xxModuleA'], // 自定义组件中需要剔除的外部模块（远程自定义组件中分享出来的模块），仅支持数组写法，需要和 remoteDeps 配合使用
  },
};
```

### 步骤 3：在组件 B 中引用

```javascript
import xxModuleA from 'xxModuleA';
```
